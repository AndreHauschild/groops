GnssParametrizationReceiver

Definition and parametrization of GNSS receivers.

Input files used in stationNetwork and lowEarthOrbiter:

  • inputfileStationInfo: Created via GnssStationLog2StationInfo or GnssStationInfoCreate.
  • inputfileAntennaDefinition: Created via GnssAntex2AntennaDefinition or GnssAntennaDefinitionCreate.
  • inputfileReceiverDefinition: Created via GnssReceiverDefinitionCreate in case you want to define which signal types a receiver model can observe.
  • inputfileAccuracyDefinition: Created via GnssAntennaDefinitionCreate.
  • inputfileSignalBias: Estimated via GnssProcessing.
  • inputfileObservation: Converted from RINEX observation files via RinexObservation2GnssReceiver.

See also GnssProcessing and GnssSimulateReceiver.

StationNetwork

A network of GNSS ground stations is defined via inputfileStationList. All input and output files except outputfileGriddedPosition, outputfileUsedStationList, inputfileAntennaDefinition, inputfileReceiverDefinition, and inputfileAccuracyDefinition are read/written for each station. The file name is interpreted as a template with the variable {station} being replaced by the station name.

If kinematicPositionEstimation=yes station positions are estimated every epoch, otherwise they are parameterized according to positionEstimation.

In case you estimate the station network together with a GNSS constellations, noNetTranslationSigma and noNetRotationSigma can be set to remove the rank deficiency. No-net constraints are only applied to stations defined in noNetStationList that have precise station coordinates provided via inputfileStationPosition. Those will be used instead of the approximate coordinates from inputfileStationInfo. In case you want to align to an ITRF/IGS reference frame, precise coordinates can be generated via combining Sinex2StationPosition, Sinex2StationDiscontinuities, and Sinex2StationPostSeismicDeformation.

A priori tropospheric correction is handled by a troposphere model (e.g. Vienna Mapping Functions 3). Additional parameters for zenith wet delay and gradients can be set up via troposphereWetEstimation and troposphereGradientEstimation. These parameters can be soft-constrained using constraints in GnssProcessing.

Antenna center variations can be estimated by defining antennaCenter. It is possible to estimate patterns for each individual station or grouped by antenna model. Antenna center parameters can be constrained by adding constraints in GnssProcessing.

The effects of loading and tidal deformation on station positions can be corrected for via deformation and tides, respectively. These typically include:

  • tides:earthTide: Earth tidal deformations (IERS conventions)
  • tides:doodsonHarmonicTide: ocean tidal deformations (e.g. FES 2014b, minDegree=1)
  • tides:doodsonHarmonicTide: atmospheric tidal deformation (e.g. AOD1B RL06, minDegree=1)
  • tides:poleTide: pole tidal deformations (IERS conventions)
  • tides:poleOceanTide: ocean pole tidal deformations (IERS conventions)

Each station goes through a preprocessing step individually, where observation outliers are removed or downweighted, continuous tracks of phase observations are defined for ambiguity parametrization, cycle slips are detected, and stations are disabled if they do not fulfill certain requirements.

LowEarthOrbiter

A single low-Earth orbiting (LEO) satellite with an onboard GNSS receiver. Orbit positions and receiver clock errors are estimated per epoch.

Attitude data must be provided via inputfileStarCamera. If no attitude data is available from the satellite operator, the star camera data can be simulated by using SimulateStarCamera or SimulateStarCameraSentinel1.

Antenna center variations can be estimated by defining antennaCenter.

In a preprocessing step observation outliers are removed or downweighted, continuous tracks of phase observations are defined for ambiguity parametrization, and cycle slips are detected.
